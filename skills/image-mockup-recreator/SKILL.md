# Image Mockup Recreator

Turn one reference image into a live HTML/CSS artifact through a measured iterate-and-score loop, not a single guess. Combines a Sequential Workflow (setup → build → publish) with an Iterative Refinement inner loop (render → score → adjust). Works from a flat image only — no Figma source needed.

## Steps

| #   | Step                           | Action                                                                                                                                                                                                                                                                                                                                                                                                               | Output / Gate                                                                                                                                                                                                                                                                           |
| --- | ------------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | Stage reference                | Copy source image to scratch dir as `target.<ext>`. Read exact pixel size (`python3 -c "from PIL import Image; print(Image.open('target.png').size)"`)                                                                                                                                                                                                                                                               | fixed viewport size — never let it drift between iterations or SSIM becomes meaningless                                                                                                                                                                                                 |
| 2   | Build first pass               | Single self-contained HTML sized exactly to target dims (`width`/`height` px on `html,body`). Absolute-position every major region (header/logo, content blocks, side panels, footer) at estimated coordinates. Don't chase fine detail yet. Avoid `filter: brightness(0) invert(1)` on colored emoji (corrupts glyphs in headless Chromium) — use inline SVG icons (`stroke="currentColor"`, `fill="none"`) instead | every region present and roughly placed                                                                                                                                                                                                                                                 |
| 3   | Render + score                 | `python3 scripts/screenshot.py <html_path> <out.png> --width <W> --height <H> [--executable-path /usr/bin/google-chrome]`<br>`python3 scripts/compare_ssim.py target.<ext> <out.png> --json --output-diff heatmap.png`                                                                                                                                                                                               | SSIM score 0-1 + verdict + up to 5 diff-region boxes (use to locate issues, not just confirm one exists); heatmap.png for showing mismatch location to user. If Playwright errors on missing browser, check `which google-chrome chromium chromium-browser` before `playwright install` |
| 4   | Iterate                        | Loop: read screenshot, compare vs target (visually + diff boxes), make **one focused category of change** per round (background shape, icons, font weight, spacing, one element's position), re-render, re-score. Score is a signal not a target — a drop can mean regression OR the metric penalizing a legit structural improvement; compare new vs previous screenshot before deciding which to keep              | stop when: score plateaus 2 rounds with no category left, OR user confirms good enough, OR ~8-10 rounds passed                                                                                                                                                                          |
| 5   | Pixel grid (if not converging) | `python3 scripts/grid_image.py target.<ext> target_grid.png --spacing 50 --color "#ff00ff"` (repeat for `<out.png>`)                                                                                                                                                                                                                                                                                                 | side-by-side grids give exact pixel offsets for relational spacing fixes — change only the specific offset/gap, don't re-derive layout                                                                                                                                                  |
| 6   | Clarify scope (if ambiguous)   | Ask whether user wants pixel/texture fidelity (real photos, gradients, font hinting) or structural fidelity (layout, spacing, correct components)                                                                                                                                                                                                                                                                    | SSIM is texture-sensitive — plateauing once structure is confirmed correct is expected when scoped to structure                                                                                                                                                                         |
| 7   | Publish                        | Publish HTML as Artifact. Republish the **same file path** on every later change                                                                                                                                                                                                                                                                                                                                     | stable URL across the whole refinement session                                                                                                                                                                                                                                          |
