# Image Mockup Recreator

Turn one reference image into a live HTML/CSS artifact through a measured iterate-and-score loop, not a single guess. Combines a Sequential Workflow (setup → build → publish) with an Iterative Refinement inner loop (render → score → adjust). Works from a flat image only — no Figma source needed.

## Steps

| #   | Step                          | Action                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              | Output / Gate                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| --- | ----------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | Stage reference               | Copy source image to scratch dir as `target.<ext>`. Read exact pixel size (`python3 -c "from PIL import Image; print(Image.open('target.png').size)"`). **Tell the user the scratch dir absolute path now** — every artifact this skill produces (target image, renders, heatmaps, grids) lands there, and they may want to open it directly                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        | fixed viewport size — never let it drift between iterations or SSIM becomes meaningless; user knows where to look                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| 2   | Build first pass              | Single self-contained HTML sized exactly to target dims (`width`/`height` px on `html,body`). Absolute-position every major region (header/logo, content blocks, side panels, footer) at estimated coordinates. Don't chase fine detail yet. Avoid `filter: brightness(0) invert(1)` on colored emoji (corrupts glyphs in headless Chromium) — use inline SVG icons (`stroke="currentColor"`, `fill="none"`) instead                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                | every region present and roughly placed                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |
| 3   | Render + score                | `python3 scripts/screenshot.py <html_path> <out.png> --width <W> --height <H> [--executable-path /usr/bin/google-chrome]`<br>`python3 scripts/compare_ssim.py target.<ext> <out.png> --json --output-diff heatmap.png`<br>`python3 scripts/compare_color.py target.<ext> <out.png> --grid 4x4 --json`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               | SSIM score 0-1 + verdict + up to 5 diff-region boxes; heatmap.png for showing mismatch location; color-diff flags regions where average color diverges beyond threshold — SSIM alone under-penalizes wrong palette (e.g. `#3B82F6` vs `#2563EB`). If Playwright errors on missing browser, check `which google-chrome chromium chromium-browser` before `playwright install`                                                                                                                                                                      |
| 4   | Iterate                       | Loop: for each flagged SSIM diff-region AND each flagged color region, crop target + render to that region's bbox and view side-by-side before deciding the fix — don't edit from a vague "looks off" impression. Make **one focused category of change** per round (background shape, icons, font weight, spacing, one element's position, one color token), re-render, re-score. Score is a signal not a target — a drop can mean regression OR the metric penalizing a legit structural improvement; compare new vs previous screenshot before deciding which to keep. **Don't swap flat gradients/placeholders for generic stock photos hoping texture will raise the score** — SSIM (grayscale structural similarity) penalizes wrong high-frequency detail harder than it penalizes flat color, so unrelated stock imagery reliably scores _worse_ than a clean gradient even though it "looks more photo-like." Only real matching source imagery helps; if unavailable, stay with gradients | stop when: score plateaus 2 rounds with no category left, OR user confirms good enough, OR ~8-10 rounds passed                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| 5   | Pixel grid (every 2-3 rounds) | `python3 scripts/grid_image.py target.<ext> target_grid.png --spacing 50 --color "#ff00ff"` (repeat for `<out.png>`). Run this proactively every 2-3 iterations, not only when stuck — waiting until "not converging" wastes rounds on eyeballed guesses that exact offsets would've caught immediately                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             | side-by-side grids give exact pixel offsets for relational spacing fixes — change only the specific offset/gap, don't re-derive layout                                                                                                                                                                                                                                                                                                                                                                                                            |
| 6   | Clarify scope (if ambiguous)  | Ask whether user wants pixel/texture fidelity (real photos, gradients, font hinting) or structural fidelity (layout, spacing, correct components)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   | SSIM is texture-sensitive — plateauing once structure is confirmed correct is expected when scoped to structure. Treat the absolute score as a floor set by available assets, not a pass/fail gate: without real matching photos, structural rebuilds commonly plateau ~0.5-0.6 even when layout is correct. Use score **deltas between rounds** and the **diff_region boxes** as the trustworthy signal (they catch real regressions/gains); don't chase a target number once structure is confirmed and the remaining gap is asset-texture only |
| 7   | Publish                       | Publish HTML as Artifact. Republish the **same file path** on every later change                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    | stable URL across the whole refinement session                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
